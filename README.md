# FPGA UART 寄存器调试项目

本项目提供了一套完整的 FPGA/ISP 寄存器调试方案，包含：

- E902 侧 UART 协议固件（寄存器读/写、版本查询）
- Python 协议库（串口封包、解包、超时与校验）
- Tkinter 图形化工具（手动读写、批量配置、功能化配置、模式切换）
- ISP 寄存器描述表（CSV 数据驱动）

适用于硬件联调、ISP 参数快速迭代、生产前配置验证等场景。

分层说明：

- 多标签页操作界面。

2. 协议层（PC）：`main.py`
- 负责 12 字节协议帧打包、发送、接收、校验与超时。
- 对 GUI 暴露统一读写接口。

3. 协议层（固件）：`main.c` / `uart_E902.c`
- 轮询式状态机解析协议。
- 执行寄存器读写并回包。

4. 驱动层：`ck_usart.c` / `ck_usart.h`
- 提供 USART 初始化、配置、收发、中断等能力。

5. 配置数据层：`ISP.csv`
- 提供寄存器模块、基地址、位域、字段描述。
- GUI 解析后动态生成配置项。

## 3. 核心模块与功能

## 3.1 固件 UART 协议模块

功能：

- 接收主机命令并解析。
- 支持寄存器读 (`CMD_READ = 0x52`)。
- 支持寄存器写 (`CMD_WRITE = 0x57`)。
- 支持版本查询 (`CMD_VERSION = 0x56`)。

关键设计：

- 采用状态机分步收包：包头 -> 包体 -> 包尾 -> 校验 -> 执行 -> 回包。
- 轮询模式下显式关闭 UART RX/TX 中断，避免 ISR 竞争 FIFO。
- 读写均使用 32-bit 绝对地址访问宏 `REG32(addr)`。

## 3.2 USART 驱动模块

功能：

- 波特率、校验位、停止位、数据位配置。
- 阻塞与非阻塞单字节收发。
- 查询式和中断式接收。

关键设计：

- 协议状态机主要依赖非阻塞收字节接口实现轮询收包。
- 驱动仍保留通用中断能力，兼容其它业务使用场景。

## 3.3 Python 协议库模块

功能：

- 定义协议帧打包与解包规则。
- 串口发送前清空输入缓冲，降低脏数据影响。
- 支持读取启动日志与固件版本解析。
- 提供 `send_fpga_command` 与批量写接口。

关键设计：

- 超时为“整体超时”而不是单次读超时，避免卡死。
- 接收侧先同步帧头，再按剩余长度读取，减少错位解析。

## 3.4 GUI 模块

主要页签功能：

1. 寄存器编辑 / 批量输入
- 手动读写寄存器。
- 32 位按位可视化与按位写回。
- CSV 批量读写导入。

2. ISP 功能配置
- 从 `ISP.csv` 解析字段生成功能卡片。
- 支持按模块、按眼别（L/R）配置。

3. 固定寄存器读写
- 固定寄存器列表一键读写。
- 支持 CSV 导入导出。

4. 模式选择
- 通过 `PAD_MUX (0x4001C030)` 配置输入模式与 MAC 输出。
- 仅修改 bit31 与 bit[3:0]，尽量不影响其它位。

## 3.5 ISP CSV 数据驱动模块

功能：

- 自动识别常见编码（`utf-8-sig`、`gb18030`、`gbk`、`utf-8`）。
- 解析模块、地址、位域、字段说明、枚举选项。
- 支持 `4001B/1D` 这类左右目紧凑基地址写法。

关键设计：

- CSV 不可用时自动回退到内置寄存器表，保证 GUI 可用性。

## 4. 协议定义

## 4.1 帧格式

固定 12 字节：

```text
Byte0   : Header   = 0x02
Byte1   : Cmd
Byte2-5 : Addr     (uint32, Big-endian)
Byte6-9 : Data     (uint32, Big-endian)
Byte10  : Checksum (sum(Byte0..Byte9) & 0xFF)
Byte11  : Tail     = 0x03
```

## 4.2 命令字

- `0x52` (`R`)：读寄存器
- `0x57` (`W`)：写寄存器
- `0x56` (`V`)：查询固件版本

## 4.3 回包规则

- 回包帧格式与请求相同。
- 读命令回包 `Data` 为寄存器当前值。
- 写命令回包通常回显写入数据。
- 版本命令回包 `Data` 编码为 `major.minor.patch`。

## 5. 快速开始

## 5.1 上位机环境

建议 Python 版本：3.8+

安装依赖：

```bash
pip install pyserial
```

说明：

- `tkinter` 通常随 Python 安装，无需单独安装。
- 若系统缺失 tkinter，请按操作系统方式补装 Tk 支持包。

## 5.2 启动 GUI

```bash
python gui.py
```

操作流程：

1. 选择串口、波特率、超时。
2. 点击“打开串口”。
3. 观察版本信息与启动日志。
4. 进入对应页签执行读写或批量配置。

## 5.3 使用协议库脚本

```python
from main import send_fpga_command

# 写寄存器
res_w = send_fpga_command(port="COM3", cmd=0x57, addr=0x4001B400, data=0x00000003)
print(res_w)

# 读寄存器
res_r = send_fpga_command(port="COM3", cmd=0x52, addr=0x4001B400, data=0)
print(res_r)
```

## 6. 固件构建与运行说明

本仓库未包含完整 SDK/工程配置脚本，通常按你们现有 E902 工程流程编译。

关键点：

1. 在 `main.c` 中启用：
- `#define WORK_MODE_TEST_UART_PROTOCOL 1`

2. 烧录固件后，串口应输出：
- `BOOT: UART protocol mode started`
- `FW_VERSION:x.y.z`

3. 确认 UART 参数与 PC 侧一致（默认 115200, 8N1）。

## 7. CSV 规范

## 7.1 手动批量读写 CSV（GUI）

写入 CSV 最少两列：

```csv
addr,data
0x4001B400,0x00000003
0x4001D400,0x00000003
```

读取 CSV 最少一列：

```csv
addr,name
0x4001B400,nlm_l
0x4001D400,nlm_r
```

支持：

- 表头别名：`addr/address/地址`
- 十六进制和十进制
- 尾缀：`U` / `L` / `UL`

## 7.2 ISP.csv 字段语义

常见列：

- `Module`：模块名
- `BaseAddress`：基地址（支持 L/R 紧凑写法）
- `Offset`：偏移
- `RegName` / `FieldName`：寄存器/字段名
- `Bits`：位域范围
- `Description`：说明与枚举

## 8. 关键设计决策

1. 协议可靠性优先
- 固件协议采用轮询状态机。
- 明确关闭 RX/TX 中断，减少收包竞争。

2. 配置可维护性优先
- 寄存器信息优先走 CSV 数据驱动。
- CSV 不可用时自动回退，保证工具可用。

3. 双目配置统一抽象
- 左右目寄存器通过基地址 + 偏移映射，减少重复配置项。

4. 操作安全性
- GUI 对范围和格式做校验。
- PAD_MUX 仅修改目标位，降低误改风险。

## 9. 常见问题（FAQ）

## 9.1 打开串口失败

检查：

- 串口号是否正确。
- 串口是否被其它工具占用。
- 用户权限是否允许访问串口。

## 9.2 有发送但无响应

检查：

- 固件是否运行在 UART 协议模式。
- 串口参数（波特率、数据位、校验位、停止位）是否一致。
- 是否存在硬件连线问题或电平不匹配。

## 9.3 版本查询失败

检查：

- 固件是否实现 `CMD_VERSION`。
- 超时参数是否过短。
- 启动阶段日志是否持续占用串口带宽。

## 9.4 ISP.csv 解析异常

检查：

- 文件编码是否异常（建议 UTF-8 with BOM 或 GB18030）。
- 列顺序是否与约定一致。
- 位域文本是否符合 `msb:lsb` 或 `bit` 格式。
